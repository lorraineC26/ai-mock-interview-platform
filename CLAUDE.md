## Project Overview

AceReady is an AI-powered mock interview platform built with Next.js 16, React 19, TypeScript, and Firebase. Users practice job interviews with a real-time AI voice interviewer (Vapi) and receive structured feedback generated by Google Gemini.

## Development Commands

```bash
# Start development server (port 3000)
npm run dev

# Build for production
npm run build

# Start production server
npm start

# Run ESLint
npm run lint
```

## Tech Stack

- **Framework**: Next.js 16.1.6 with App Router
- **React**: 19.2.3 with React Compiler enabled
- **Styling**: Tailwind CSS 4 with custom CSS variables and animations
- **Database & Auth**: Firebase (Firestore for data, Firebase Auth + session cookies)
- **Voice Agent**: Vapi (`@vapi-ai/web` v2.5.2) for real-time AI interviewer
- **AI/LLM**: Google Gemini 2.5-flash-lite via `@ai-sdk/google` + Vercel `ai` SDK
- **UI Components**: Radix UI primitives, shadcn/ui patterns
- **Forms**: React Hook Form with Zod validation
- **Icons**: Lucide React
- **Date Handling**: Day.js
- **Notifications**: Sonner for toast messages

Two separate Firebase configurations:

- **`firebase/client.ts`** вЂ” Client-side Firebase (auth, Firestore). Uses `NEXT_PUBLIC_*` env vars.

### Server Actions (`lib/actions/`)

**`auth.action.ts`** вЂ” All marked `"use server"`
- `signUp(params)` вЂ” Creates user in Firestore; prevents duplicates
- `signIn(params)` вЂ” Verifies user, calls `setSessionCookie`
- `setSessionCookie(idToken)` вЂ” Creates httpOnly session cookie (7-day expiry, sameSite=lax)
- `getCurrentUser()` вЂ” Reads session cookie, verifies with Admin SDK, returns user from Firestore
- `isAuthenticated()` вЂ” Returns `!!getCurrentUser()`

**`general.action.ts`**
- `getInterviewsByUserId(userId)` вЂ” User's own interviews (ordered by `createdAt` desc)
- `getLatestInterviews({ userId, limit? })` вЂ” Finalized interviews from OTHER users (for practice)
- `getInterviewById(id)` вЂ” Single interview by ID
- `createFeedback(params)` вЂ” Calls Gemini with transcript, generates structured feedback, saves to Firestore
- `getFeedbackByInterviewId({ interviewId, userId })` вЂ” Fetches feedback by `interviewId` + `userId`

### API Routes

**`POST /api/vapi/generate`** вЂ” Called by Vapi workflow after collecting interview parameters
- Accepts: `{ type, role, level, techstack, amount, userid }`
- Calls Gemini to generate `amount` interview questions as a JSON array
- Saves new interview document to Firestore with `finalized: true`

### AI Integration

**Gemini (`gemini-2.5-flash-lite`)** is used in two places:
1. **Question generation** (`/api/vapi/generate`) вЂ” Generates a JSON array of interview questions tailored to role/level/techstack. Avoids `/` and `*` for voice compatibility.
2. **Feedback generation** (`createFeedback`) вЂ” Analyzes the interview transcript and returns structured feedback via Vercel AI SDK's `generateText` with `Output.object()`. Five scoring categories: Communication Skills, Technical Knowledge, Problem-Solving, Cultural & Role Fit, Confidence & Clarity (each 0вЂ“100 with comment).

**Vapi** handles real-time voice interaction. The workflow (`assets/vapi_workflow.json`) collects interview parameters via conversation, then calls `/api/vapi/generate`. The SDK is initialized in `lib/vapi.sdk.ts`.

## Key Implementation Notes

### React Compiler
- React Compiler is enabled (`reactCompiler: true` in `next.config.ts`)
- Follow React best practices to maximize automatic optimization

### Authentication Flow
- Firebase Auth creates the user; a separate Firestore `users` document stores profile data
- Session cookie (httpOnly, 7-day, sameSite=lax) is used for server-side auth checks
- Both `(root)` and `(auth)` layouts call `isAuthenticated()` to redirect appropriately
- **Important:** Firestore uses `userID` (capital ID) on interview documents; feedback uses `userId` (lowercase d) вЂ” do not confuse these field names

### Interview Data Flow
1. User visits `/interview` в†’ `Agent type="generate"` starts Vapi workflow
2. Vapi collects params, calls `/api/vapi/generate` в†’ Gemini generates questions в†’ saved to Firestore
3. User visits `/interview/[id]` в†’ `Agent type="interview"` conducts voice interview
4. On call end в†’ `createFeedback()` sends transcript to Gemini в†’ feedback saved to Firestore
5. User redirected to `/interview/[id]/feedback` to view results

### Dashboard Layout
- Home page fetches user's own interviews and other users' interviews in parallel (`Promise.all`)
- "Your Interviews" section shows past attempts with scores
- "Take an Interview" section shows other users' finalized interviews to practice on

### Styling
- **Dark-only theme**: `<html className="dark">` enforced in root layout; no light mode
- **Custom CSS variables**: Primary colors (`#dddfff`, `#cac5fe`), dark backgrounds (`#020408`, `#27282f`, `#242633`)
- **Path alias**: `@/` maps to project root
- **Font**: Mona Sans from Google Fonts

### Date Formatting
- Day.js throughout. Standard format: `"MMM D, YYYY"` (e.g., `"Feb 19, 2024"`)
